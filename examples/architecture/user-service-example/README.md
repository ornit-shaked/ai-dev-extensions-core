## 💡 How to Use This Example

### For Developers

**Learning the format**:
1. Start with `1-identity.md` to understand service scope
2. Review `_metadata.yaml` to see completeness tracking
3. Check `_open-issues.md` for typical gaps

**Comparing your output**:
- Your workflow output should match this structure
- Metadata format should be consistent
- Open issues should follow same priority system

### For AI Agents

**Understanding what to expect**:
1. Read `_metadata.yaml` first for quick assessment
2. Check `agent_ready` status and confidence level
3. Use section metadata to gauge reliability
4. Respect `needs_human: true` flags

**Example query flow**:
```
Agent task: "Add new API endpoint for user search"

1. Read _metadata.yaml
   → agent_ready: PARTIAL
   → contracts section: needs_human: true
   
2. Read 3-contracts.md
   → See existing endpoints
   → Note OpenAPI spec location unverified
   
3. Read 5-flows.md
   → Understand user search flow
   
4. Proceed with caution
   → Verify OpenAPI spec before implementation
   → Follow existing patterns
```

## 🎓 Learning Points

### Metadata Patterns

**COMPLETE** sections:
- All expected information found
- High confidence
- Safe for agent use

**PARTIAL** sections:
- Core information present
- Some details missing
- Use with caution
- Check open issues

**Source indicators**:
- **AUTO**: Machine-extracted (high confidence)
- **SEMI**: Mixed automation + validation needed
- **MANUAL**: Requires human input

### Risk Levels

**LOW**: Safe to modify with agent assistance  
**MEDIUM**: Review changes carefully  
**HIGH**: Critical section, human review mandatory
